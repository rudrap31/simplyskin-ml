# ML output schema — `detector_landmarks_v1` (FROZEN)

Produced by `src/inference/detector_landmarks_pipeline.py::run_pipeline_on_image`.
The `pipeline_version` field in every result is the schema version — bump
it (e.g. `detector_landmarks_v2`) if this shape ever changes; do not
silently change field meaning under the same version string.

Full example: [`example_result_detector_landmarks_v1.json`](example_result_detector_landmarks_v1.json)
— a real, unmodified pipeline output (not hand-edited), generated by
`python3 src/scripts/run_detector_landmarks_pipeline.py --image <path>`.

## Region naming (read this before wiring up the backend)

Region names — `forehead`, `image_left_cheek`, `image_right_cheek`,
`nose`, `chin`, `other_face`, and `unassigned` — are from the **viewer/
image coordinate perspective**, not the subject's anatomical left/right.
`image_left_cheek` is whatever appears on the left side of the photo as
displayed; it is mirrored relative to the subject's own left cheek in a
typical selfie.

- This repo does not attempt subject-anatomical left/right conversion.
  If the backend needs "your left cheek" wording, it must do that
  conversion itself, using its own knowledge of camera/mirroring context
  (which this repo cannot infer from pixels alone).
- **Input image orientation must be normalized before calling this
  pipeline** (EXIF-rotated upright, not pre-mirrored) — the pipeline
  assumes pixel (0,0) is the top-left of the image as a human would view
  it. Feed it a mirrored or sideways frame and `image_left_cheek` will
  faithfully describe the wrong physical side of the subject's face.

## Top-level fields

| field | type | notes |
|---|---|---|
| `pipeline_version` | string | schema version, currently `"detector_landmarks_v1"` |
| `face_detected` | bool | |
| `multiple_faces_detected` | bool | true if >1 face found; largest/most-central was used, see `quality.warnings` |
| `image.width` / `image.height` | int | source image pixel dimensions |
| `detections` | list | empty if no face or no detections above threshold |
| `summary` | object | aggregate stats, see below |
| `quality.landmark_confidence` | always `null` | MediaPipe's Tasks API doesn't expose a single scalar confidence for Face Landmarker |
| `quality.warnings` | list of strings | e.g. no face, multiple faces, region assignment skipped |
| `models.detector` / `models.face_landmarker` | string | model identity/version for traceability |

## `detections[]` fields

`box` (pixel xyxy), `normalized_box` (0–1 xyxy), `confidence`, `region`
(one of the 6 region names or `"unassigned"`), `region_overlap_ratio`,
`relative_face_area` (box area / face-oval area).

## `summary` fields

`total_lesions`, `assigned_lesions`, `unassigned_lesions`,
`average_detection_confidence`, `counts_by_region` (6 region keys),
`dominant_region`, `cheek_count` (image_left + image_right),
`t_zone_count` (forehead + nose + chin),
`image_left_right_cheek_difference` (image_left_cheek count − image_right_cheek count).

## Region assignment rules (summary — full detail in `src/landmarks/regions.py::assign_detection_to_region`)

1. Compute overlap ratio (intersection area / detection box area) against
   each of the 6 region polygons.
2. If the best region's ratio ≥ 0.10, assign it (`assignment_method: "overlap"`).
3. Otherwise, check the box's center point: if it falls inside an excluded
   zone (eyes/eyebrows/lips) or outside the face oval, mark `unassigned`.
4. Otherwise, assign whichever region polygon contains the center point
   (`assignment_method: "centroid_fallback"`).
